# Repository Guidelines

## Project Structure & Module Organization

- `src/index.ts` contains the CLI entry point, Koa server, asset rewriting, and proxy logic.
- `dist/` is generated by TypeScript and is the published binary target from `package.json`; do not edit it by hand.
- `docs/` stores README assets such as screenshots.
- Root config files include `package.json`, `tsconfig.json`, `.eslintrc.yaml`, `.npmrc`, and npm ignore settings.

Keep runtime code in `src/`. If behavior grows beyond one file, split by responsibility, for example `src/proxy.ts`, `src/assets.ts`, or `src/args.ts`.

## Build, Test, and Development Commands

- `npm install`: install runtime and development dependencies.
- `npm run prepare`: run `tsc` and emit `dist/` plus declarations.
- `npm run watch`: run TypeScript in watch mode.
- `npm run eslint`: lint tracked `.ts` and `.tsx` files with `@laverdet/eslint-config`.
- `npx screeps-steamless-client --package /path/to/Screeps/package.nw`: start the CLI.

The package defaults to `localhost:8080`. Use `--host`, `--port`, `--backend`, and `--internal_backend` when validating private server behavior.

## Coding Style & Naming Conventions

Use TypeScript ES modules and strict typing. The current style uses tabs, single quotes, semicolons, and concise arrow functions. Prefer `const` unless reassignment is required. Keep command-line options in long `--kebab` form, or preserve existing underscore flags such as `--internal_backend`.

Avoid broad rewrites in `src/index.ts`; its behavior is coupled to Screeps client quirks. Add short comments only for non-obvious compatibility patches or protocol behavior.

## Testing Guidelines

There is no automated test suite yet. Run `npm run prepare` and `npm run eslint` before submitting changes. Manually smoke test the CLI against at least one target:

- Official server: `http://localhost:8080/(https://screeps.com)/`
- Local server: `http://localhost:8080/(http://localhost:21025)/`

When adding tests, place them under `test/` or `src/**/*.test.ts`. Cover URL extraction, asset rewriting, and proxy request rewriting first.

## Commit & Pull Request Guidelines

Recent commits use short, imperative subjects, sometimes with a prefix such as `fix:`. Examples: `fix: private server compatibility`, `Add backend and host args`.

Pull requests should include a concise description, the server/client scenario tested, and any changed CLI flags or URL formats. Link related issues. Include screenshots only for visible client changes.

## Security & Configuration Tips

Do not commit local Steam paths, tokens, cookies, or generated client packages. Treat proxied backend URLs as user input and preserve existing origin, redirect, and WebSocket handling unless the change explicitly targets those paths.
